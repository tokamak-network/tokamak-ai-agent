# 🗺️ Tokamak AI Agent 개발 로드맵 (Roadmap)

> **작성 일자**: 2026-02-23
> **현재 상태**: v0.1.2 기반 (기본 Agent, Chat, Terminal 연동 부분 완료)

본 문서는 Tokamak AI Agent의 지속적인 고도화와 진정한 '사내 1:1 페어 프로그래밍 AI'로 거듭나기 위한 중장기적인 개발 로드맵 및 백로그(Backlogs)를 새롭게 제안하는 문서입니다.

현재 완성도 높은 Agent 및 Chat 시스템을 바탕으로, 아래 기능들을 순차적으로 도입하여 사용자 경험(UX)과 문제 해결 자율성(Autonomy)을 극대화합니다.

---

## 🎯 Phase 1: 개발 흐름 연동 및 디버깅 자동화 (단기)

가장 먼저 실무 개발자들이 에디터와 터미널에서 겪는 "작성 - 실행 - 에러 - 수정"의 반복 타이핑과 윈도우 스위칭을 완벽히 줄여줍니다.

### 1. 🖥️ 고급 에러 트래킹 및 자동 수정 루프 (Enhanced Terminal Loop)
- **실시간 에러 스트리밍 분석**: 사용자가 띄워둔 장시간 프로세스(dev server 구동 중)에서 에러가 났을 때 Agent가 이를 실시간으로 분석해 알림 및 "에러 원인 분석/수정(Auto-Fix)" 제안.
- **자율 디버깅 완전화(Closed-Loop)**: Agent 자체가 자체 터미널 환경을 백그라운드로 제어하면서, 테스트 코드 통과 여부를 검증하고 에러를 반복 자동 수정.

### 2. 🔀 Git 및 협업 자동화 (VCS Integration)
- **Commit 메시지 생성**: 현재 작업 중인 파일의 `git diff`를 분석하여 커밋 메시지 초안 자동 생성 (패널 내 원클릭 커밋 버튼 제공).
- **PR(Pull Request) 초안 작성**: 에이전트가 기획/구현한 코드의 맥락을 요약하여 PR 설명 마크다운 작성 보조.

## 🚀 Phase 2: 인지 모델의 한계 돌파 및 지식 구축 (중기)

단일 파일, 특정 폴더를 넘어 프로젝트 전체의 맥락을 이해하고, 사용자의 코딩 습관을 지속적으로 흡수합니다.

### 1. 🧠 코드베이스 전체 시맨틱 검색 (AST & RAG 고도화)
- **의미 기반 검색(Semantic Search)**: "결제 관련된 모듈 찾아서 분석해줘" 같은 추상적 질문에 대해 벡터 DB나 AST 파싱을 활용해 연관된 전체 파일 그래프를 AI 컨텍스트에 포함.
- **대규모 리팩토링 스캐너**: 대형 사내 프로젝트에서 함수명 하나를 바꿀 때 전체 의존성을 훑고 관련 모듈을 모두 찾아 제시하는 기능 구축 (단순 문자열 기반 `grep` 이상의 의미론적 인지).

### 2. 📚 자율 학습형 개인화 메모리 (Auto-Learning Memory)
- **현재 구현된 지식 베이스와의 차이점**: 현재 구현된 `.tokamak/knowledge/` 폴더는 사용자가 명시적으로 마크다운 문서를 작성해 두는 **'수동/프로젝트 종속적'** 지식 베이스입니다. 반면 이 제안 기능은 AI가 대화 속에서 스스로 학습하는 **'자동/개인 종속적'** 메모리입니다.
- **점진적 행동 학습**: 대화 도중 "나는 세미콜론 안 써", "이 프로젝트는 axios 말고 fetch를 써"라고 힌트를 주거나, AI 스스로 반복적인 코드 패턴/선호도를 파악하면 이를 익스텐션의 영구 저장소에 자동 기록(Self-updating)하고 이후 컨텍스트에 알아서 반영합니다.

### 3. 직관적인 Diff 리뷰 UI (Human-in-the-Loop)
- 에이전트가 여러 코드를 한 번에 수정했을 때, 체크포인트를 넘어 VS Code 본연의 **Diff 창(변경 전/후 병렬 구조)** 을 적극 연동하여, 사용자가 한 눈에 직관적으로 코드를 병합 및 수락(Accept)/거절(Reject) 할 수 있는 모달 UI 고도화.

## 🌌 Phase 3: 무한한 확장성과 특수 환경 대응 (장기)

### 1. 👥 사내 Skill 파편화 해결 (Marketplace 연동)
- **중앙화된 Skill Hub**: 현재의 `.tokamak/skills/` 폴더 기반 로컬 관리 구조를 넘어, 사내 서버/Git API를 연동한 전사적 중앙 "프롬프트 마켓플레이스" 구축.
- **팀 단위 구독**: 훌륭한 리뷰 프롬프트나 템플릿 제너레이터를 팀원들이 버튼 하나로 구독·동기화할 수 있도록 지원.

### 2. 👁️ UI/프론트엔드 검증 에이전트 (Browser Automation)
- 단순 백엔드/문서 수정을 넘어 Playwright, Puppeteer와 같은 헤드리스 브라우저를 백그라운드로 구동시켜 작성한 React/Vue 코드가 실제로 어떻게 렌더링되었는지 **시각적 DOM 및 스크린샷 캡처** 후 결과 검증 수행.

### 3. 멀티 에이전트 아키텍처 (Team of Agents)
- 특정 태스크를 해결하기 위해 `Plan Agent`, `Review Agent`, `Code Agent` 등 역할을 분담한 다중 에이전트가 상호 피드백과 검증을 거쳐 최종 결과물만 유저에게 전달하는 고도화된 스웜(Swarm) 구조 설계.
